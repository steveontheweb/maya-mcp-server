## Обзор

Maya MCP Server позволяет Claude AI напрямую управлять Autodesk Maya через Model Context Protocol, предоставляя:

- 🎨 3D-моделирование с помощью ИИ
- 🤖 Управление Maya на естественном языке
- 📸 Предварительный просмотр сцены в реальном времени
- 🔧 Автоматизация рабочего процесса

## Быстрый Старт

### 1. Установка Зависимостей

```bash
pip install "mcp[cli]>=1.3.0"
```

### 2. Загрузка Плагина Maya

Загрузите `plug-ins/maya_mcp.py` в **Менеджере плагинов** Maya:

1. Скопируйте `plug-ins/maya_mcp.py` в:
   - Windows: `C:\Users\<имя_пользователя>\Documents\maya\<версия>\plug-ins\`
   - macOS: `~/maya/<версия>/plug-ins/`

2. Откройте Maya → **Windows > Settings/Preferences > Plug-in Manager**

3. Найдите `maya_mcp.py`, отметьте **Loaded** и **Auto load**

4. Проверьте в редакторе скриптов:
   ```
   MayaMCP: Плагин успешно загружен
   MayaMCP: Сервер запущен на localhost:9877
   ```

### 3. Настройка Claude Desktop

Отредактируйте файл конфигурации (**Settings > Developer > Edit Config**):

#### Метод A: Использование npx (Рекомендуется)

```json
{
  "mcpServers": {
    "maya-mcp": {
      "command": "npx",
      "args": [
        "--yes",
        "--package=путь_к_проекту",
        "maya-mcp"
      ],
      "env": {
        "MAYA_HOST": "localhost",
        "MAYA_PORT": "9877"
      },
      "alwaysAllow": [
          "get_scene_info",
          "create_primitive",
          "delete_object",
          "set_material",
          "get_viewport_screenshot",
          "transform_object",
          "smart_select",
          "get_scene_summary",
          "get_console_output",
          "execute_maya_code"
      ]
    }
  }
}
```

#### Метод B: Использование Python

```json
{
  "mcpServers": {
    "maya-mcp": {
      "command": "python",
      "args": ["-m", "maya_mcp.server"],
      "env": {
        "PYTHONPATH": "путь_к_проекту/src",
        "MAYA_HOST": "localhost",
        "MAYA_PORT": "9877"
      },
      "alwaysAllow": [
          "get_scene_info",
          "create_primitive",
          "delete_object",
          "set_material",
          "get_viewport_screenshot",
          "transform_object",
          "smart_select",
          "get_scene_summary",
          "get_console_output",
          "execute_maya_code"
      ]
    }
  }
}
```

> 💡 Дополнительные методы конфигурации смотрите в каталоге [`examples/`](examples/)

### 4. Проверка Соединения

Перезапустите Claude Desktop и спросите:

```
Получить информацию о текущей сцене Maya
```

Если вы видите информацию о сцене, соединение успешно! ✅

## Возможности

### 🛠️ Основные Инструменты

| Инструмент | Функция |
|-----|------|
| `get_scene_info` | Получить информацию о сцене (объекты, камеры, освещение) |
| `get_object_info` | Получить детали объекта (положение, вращение, материал) |
| `create_primitive` | Создать геометрию (cube/sphere/cylinder/plane/torus) |
| `delete_object` | Удалить объект |
| `transform_object` | Трансформировать объект (переместить/повернуть/масштабировать) |
| `set_material` | Установить материал и цвет |
| `execute_maya_code` | Выполнить Python-код |
| `get_viewport_screenshot` | Захватить скриншот области просмотра ⚠️ |
| `smart_select` | Умный выбор объектов с regex и фильтрами |
| `get_scene_summary` | Получить полную сводку сцены |
| `get_console_output` | Получить вывод консоли/редактора скриптов Maya 🆕 |

> ⚠️ Примечание: `get_viewport_screenshot` может быть нестабильным в некоторых версиях Maya из-за проблем совместимости с playblast.

## Параметры Конфигурации

### Переменные Окружения

| Переменная | Значение по умолчанию | Описание |
|------|--------|------|
| `MAYA_HOST` | `localhost` | Адрес сервера Maya |
| `MAYA_PORT` | `9877` | Порт сервера Maya |
| `PYTHONPATH` | - | Путь поиска модулей Python (требуется только для прямого режима Python) |

### Пользовательский Порт

**Изменить в плагине Maya:**
```python
start_maya_mcp_server(host='localhost', port=9878)
```

**Изменить в файле конфигурации:**
```json
"env": {
  "MAYA_PORT": "9878"
}
```

## Устранение Неполадок

### Ошибка Подключения

**Проблема:** "Невозможно подключиться к Maya"

**Решение:**
1. ✅ Убедитесь, что Maya запущена
2. ✅ Убедитесь, что плагин загружен (проверьте Менеджер плагинов)
3. ✅ Проверьте сообщение о запуске в редакторе скриптов
4. ✅ Убедитесь, что порт 9877 не используется

### Модуль Не Найден

**Проблема:** "ModuleNotFoundError: No module named 'maya_mcp'"

**Решение:**
1. Установите зависимости: `pip install "mcp[cli]>=1.3.0"`
2. Проверьте настройку PYTHONPATH (при использовании прямого режима Python)
3. Попробуйте использовать метод npx

### Ошибка Загрузки Плагина

**Проблема:** Maya сообщает "No initializePlugin() function"

**Решение:**
- Убедитесь, что используете последнюю версию `maya_mcp.py`
- Файл плагина должен содержать функции `initializePlugin()` и `uninitializePlugin()`

## Предупреждения Безопасности

⚠️ **Важное Примечание:**

- Инструмент `execute_maya_code` позволяет выполнять произвольный Python-код
- Всегда **сохраняйте сцену Maya** перед выполнением
- Используйте с осторожностью в производственной среде
- Рекомендуется сначала проверять операции в тестовых сценах

## Журналы

Журналы сервера сохраняются в:
- **Windows:** `%TEMP%\maya-mcp\maya-mcp.log`
- **macOS/Linux:** `/tmp/maya-mcp/maya-mcp.log`

Просмотрите журналы для отладки проблем с подключением или ошибок выполнения команд.

## Разработка

### Добавление Новых Инструментов

1. Добавьте обработчик команд в `plug-ins/maya_mcp.py`
2. Добавьте определение инструмента MCP в `src/maya_mcp/server.py`
3. Протестируйте новый инструмент

### Запуск Сервера Разработки

```bash
# Настроить окружение
export PYTHONPATH=/path/to/maya-mcp-server/src

# Запустить сервер
python -m maya_mcp.server
```
